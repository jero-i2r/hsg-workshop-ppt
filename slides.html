如何分辨项目难度 (I)

ANALYSIS
我想做一个[描述你的项目]，[自己用/内部分享/在xx地区面对xx用户公开/商用]。我完全没有编程基础，打算用 AI 辅助完成（vibe coding）。请帮我评估： 1. 这个项目的核心功能拆解成哪些模块？ 2. 每个模块的技术难度（简单/中等/困难）？ 3. 哪些部分 AI 可以直接生成代码搞定，哪些需要人工调试？ 4. 有没有现成的模板、框架或开源项目可以直接用？ 5. 零基础的人最容易卡在哪里？
WORKLOAD
基于上面的功能拆解，假设我用 [工具名字如Claude] 做 vibe coding： 1. 每个模块大概需要多少轮 AI 对话才能完成？ 2. 整个项目从零到能跑起来，大致需要多少小时？ 3. 最小可用版本（MVP）只保留哪些功能？做 MVP 要多久？ 4. 有没有什么功能看起来简单但实际上是"深坑"？

如何分辨项目难度 (II)

TECH STACK
我想要确认一下技术路线。对于这个项目，请推荐一套对零基础最友好的技术栈： 1. 前端用什么？后端用什么？数据库用什么？ 2. 部署到哪里最简单（最好免费）？ 3. 这套技术栈在 AI 代码生成中的支持度如何？
RISK CHECK
风险排雷。有没有"隐形门槛"： 1. 需要第三方 API 且配置复杂的（支付/OAuth）？ 2. 需要处理实时数据或高并发的？ 3. 涉及用户隐私/安全合规的？ 4. 需要复杂的部署配置（Docker、CI/CD 等）？
ADVANCED TIPS
进阶技巧： 角色提示。如：请你扮演一个有 10 年经验的全栈工程师，对我坦诚评估。不要为了鼓励我而低估难度。 用具体场景代替模糊描述。如：不要说"我想做一个社交 APP"，而是说："我想做一个小工具：用户可以上传一张照片，AI 自动生成一段描述文字...不需要登录，不需要评论功能。"
